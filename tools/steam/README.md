# Steam upload — Playtest

A Steam **Playtest** is the friction-free way to get builds to testers, including non-technical
ones. Steam downloads content through its own client (so files are **not** Gatekeeper-quarantined)
and launches the game binary **directly**, so the macOS build runs with **no notarization and no
Gatekeeper prompts** — the tester installs Steam and clicks **Play**. One Playtest delivers the
Windows, macOS, and Linux builds together.

IDs used here:
- Playtest **App ID `4933640`** (the normal app is `4897210` — used later for the public store page)
- Depot **`4933641`** (the one your packages reference) holds all three OS builds

## Build + upload

```
./run.sh dist steam          # builds all 3 → dist/steam/content/{windows,macos,linux}
cd tools/steam
steamcmd +login <steam_user> +run_app_build "$(pwd)/app_build.vdf" +quit
```

`steamcmd` will prompt for your password + Steam Guard code — it must run on a machine signed into
your Steam account (can't be automated/headless the first time). After it finishes, in Steamworks →
the **Playtest** app → **SteamPipe → Builds**, set the new build live on the playtest branch (or set
`"setlive" "playtest"` in `app_build.vdf`).

## Launch options (Playtest app → Installation → General → Launch Options)

Add one per OS (set the **Operating System** filter on each):

| OS      | Executable                        |
|---------|-----------------------------------|
| Windows | `windows/toomanymachines.exe`     |
| macOS   | `macos/TooManyMachines.app`       |
| Linux   | `linux/toomanymachines`           |

(One depot means every tester downloads all three OS folders — fine for a playtest. To shrink
downloads later, make one depot per OS with each depot's OS filter set, and reference all three
in the playtest package.)

## Art

Upload in the dashboard (you can do this from a phone browser). Files come from `dist/steam/art/`,
generated by:

```
./run.sh shot                                   # writes .debug_screenshots/vk.jpg
python3 tools/steam/make_placeholder_art.py     # → dist/steam/art/*
```

For the **Playtest** specifically you need little — mainly its **header capsule** + **app icon**:

| file                          | slot                                            |
|-------------------------------|-------------------------------------------------|
| `playtest_header_460x215`     | Playtest → Store capsule → Header               |
| `app_icon_184x184`            | Community → App Icon                             |
| `shortcut_icon.ico`           | Community → Shortcut Icon                        |
| `screenshot_01_1280x720.jpg`  | Store → Screenshots (add a few)                 |

The rest (small/main/vertical capsules, library capsule/hero/logo, page background) are for the
**main** app's "Coming Soon"/release page later. All are placeholders built from a real gameplay
frame — swap in real key art anytime.

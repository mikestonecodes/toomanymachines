# Steam upload (Playtest)

A Steam **Playtest** is the friction-free way to get builds to testers — including
non-technical ones. Steam downloads content through its own client (so files are **not**
Gatekeeper-quarantined) and launches the game binary **directly**, so the macOS build runs
with **no notarization and no Gatekeeper prompts**: the tester just installs Steam and clicks
**Play**. The same Playtest delivers the Windows and Linux builds too.

## One-time setup (Steamworks)

1. Join the Steamworks program and pay the one-time **$100 Steam Direct** fee; create your app.
2. On the app's landing page → **Manage → Enable Playtest**. This creates a *second* app (the
   Playtest) with its own App ID. Testers request access; you approve them.
3. Note the IDs you'll need:
   - the **main App ID**
   - one **Depot ID** per OS (Steamworks → SteamPipe → Depots — add three: Windows, macOS, Linux).
4. Fill those into the `.vdf` files here (replace every `YOUR_..._ID`):
   - `app_build.vdf` — App ID + the three depot IDs
   - `depot_windows.vdf`, `depot_macos.vdf`, `depot_linux.vdf` — their depot IDs

## Build + upload a new build

```
./run.sh dist steam          # builds all 3 targets → dist/steam/content/{windows,macos,linux}
cd tools/steam
steamcmd +login <steam_user> +run_app_build "$(pwd)/app_build.vdf" +quit
```

Then in the Steamworks dashboard → **SteamPipe → Builds**, set the new build live on the
Playtest branch (or set `"setlive" "playtest"` in `app_build.vdf` to do it automatically).

## Launch options (set once, in the web dashboard)

Steamworks → **Installation → General → Launch Options** (add one per OS):

| OS      | Executable                                  |
|---------|---------------------------------------------|
| Windows | `toomanymachines.exe`                       |
| macOS   | `TooManyMachines.app`                        |
| Linux   | `toomanymachines`                           |

## Store / Playtest art

Steam requires capsule art + screenshots before the page can go live. Placeholders (built from a
real gameplay frame) are generated by:

```
./run.sh shot                                   # writes .debug_screenshots/vk.jpg
python3 tools/steam/make_placeholder_art.py     # → dist/steam/art/*
```

`dist/steam/art/` then holds, at the exact Steam sizes:

| file                          | where it goes in Steamworks                     |
|-------------------------------|-------------------------------------------------|
| `header_capsule_460x215`      | Store capsule → Header capsule                  |
| `small_capsule_231x87`        | Store capsule → Small capsule                   |
| `main_capsule_616x353`        | Store capsule → Main capsule                    |
| `vertical_capsule_374x448`    | Store capsule → Vertical capsule                |
| `library_capsule_600x900`     | Steam library → Capsule                         |
| `library_hero_1920x620`       | Steam library → Hero                            |
| `library_logo_1280x720`       | Steam library → Logo (transparent)             |
| `page_background_1438x810`    | Store page background                           |
| `playtest_header_460x215`     | the Playtest app's Header capsule               |
| `screenshot_01_1280x720.jpg`  | Store → Screenshots (add ≥5; capture more via `./run.sh shot`) |

These are stand-ins so you can submit now; swap in real key art any time. For more screenshots,
re-run `./run.sh shot` (edit the drive in `tools/devharness` to vary ride/weapon/camera), or just
press the screenshot key in-game once it's on Steam.
